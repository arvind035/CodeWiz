We use offline, small but well capable coding language models, to run through full project repositories or merge request changes to analyze and understand the code base.

These models go through modules and suggest optimizations, discover potential bugs and document in layman terms on-the-go. Finally we put everything together with another offline LLM and create a whole guidebook for the project in simple English.

This can be used to make the modules better or just for understanding a legacy project in a few minutes. Everything is done keeping the codebase to our own servers, without sharing any IP with any third party API.
